Color Contrast (Level AAA)\n- **Minimum ratio**: 4.5:1 for normal text\n- **Maximum ratio**: 7:1 recommended for body text\n- **Tools**: WebAIM Contrast Checker, aXe DevTools\n\n**Current Implementation**:\n```css\n/* Primary brand on white - 5.1:1 contrast */\ncolor: #de3b28;  /* primary-600 */\nbackground-color: #ffffff;\n\n/* Verified in color palette configuration */\n```\n\n#### 2. Focus Indicators\n```css\n/* Keyboard navigation support */\nbutton:focus-visible,\na:focus-visible {\n  outline: 3px solid var(--color-primary-600);\n  outline-offset: 2px;\n}\n```\n\n#### 3. Semantic HTML\n```jsx\n// Good: Semantic landmarks\n<header role=\"banner\">...</header>\n<nav aria-label=\"Main navigation\">...</nav>\n<main role=\"main\">...</main>\n<aside role=\"complementary\">...</aside>\n<footer role=\"contentinfo\">...</footer>\n\n// Good: Heading hierarchy\n<h1>Page Title</h1>\n<h2>Section</h2>\n<h3>Subsection</h3>\n```\n\n#### 4. ARIA Labels\n```jsx\n// Icon buttons must have labels\n<button aria-label=\"Close menu\" onClick={closeMenu}>\n  <svg>...</svg>\n</button>\n\n// Form fields with labels\n<label htmlFor=\"email\">Email</label>\n<input id=\"email\" type=\"email\" />\n\n// Live regions\n<div role=\"status\" aria-live=\"polite\" aria-atomic=\"true\">\n  {loadingMessage}\n</div>\n```\n\n#### 5. Screen Reader Support\n```jsx\n// Hidden but readable by screen readers\n<span className=\"sr-only\">Loading...</span>\n\n// Skip to main content (hidden on desktop)\n<a href=\"#main\" className=\"skip-to-main\">\n  Skip to main content\n</a>\n```\n\n#### 6. Keyboard Navigation\n```jsx\n// All interactive elements are keyboard-accessible\nconst Card = ({ clickable, onClick }) => (\n  <div\n    role={clickable ? 'button' : undefined}\n    tabIndex={clickable ? 0 : undefined}\n    onKeyDown={(e) => {\n      if (e.key === 'Enter' || e.key === ' ') {\n        onClick?.(e);\n      }\n    }}\n  >\n    {/* Content */}\n  </div>\n);\n```\n\n### Accessibility Utilities\n\n```css\n/* Screen reader only text */\n.sr-only {\n  position: absolute;\n  width: 1px;\n  height: 1px;\n  padding: 0;\n  margin: -1px;\n  overflow: hidden;\n  clip: rect(0, 0, 0, 0);\n  white-space: nowrap;\n  border-width: 0;\n}\n\n/* Reduce motion for users with vestibular disorders */\n@media (prefers-reduced-motion: reduce) {\n  * {\n    animation-duration: 0.01ms !important;\n    transition-duration: 0.01ms !important;\n  }\n}\n```\n\n---\n\n## ⚡ Performance Optimization\n\n### 1. Image Optimization\n\n#### Lazy Loading Strategy\n```jsx\n<LazyImage\n  src=\"image.jpg\"\n  loading=\"lazy\"           // Native lazy loading\n  sizes=\"(max-width: 480px) 100vw, 50vw\"\n  width={1200}\n  height={800}\n  priority={false}         // Only eager for critical images\n/>\n```\n\n#### Responsive Image Sizes\n```html\n<!-- Automatic resolution switching -->\n<img\n  srcset=\"image-320w.jpg 320w, image-640w.jpg 640w, image-1200w.jpg 1200w\"\n  sizes=\"(max-width: 480px) 320px, (max-width: 768px) 640px, 1200px\"\n  alt=\"Description\"\n/>\n```\n\n### 2. Core Web Vitals Optimization\n\n#### LCP (Largest Contentful Paint) < 2.5s\n- Preload critical images\n- Optimize server response time\n- Minimize CSS/JS\n\n#### FID (First Input Delay) < 100ms\n- Break up long JavaScript tasks\n- Use Web Workers for heavy processing\n- Defer non-critical scripts\n\n#### CLS (Cumulative Layout Shift) < 0.1\n```css\n/* Always specify dimensions to prevent shifts */\n.image-container {\n  aspect-ratio: 16 / 9;  /* Reserve space before image loads */\n}\n\nimg {\n  width: 100%;\n  height: auto;\n}\n```\n\n### 3. CSS/JS Optimization\n\n#### Bundle Size Reduction\n```bash\n# Minify production bundles\nterser input.js -c -m -o input.min.js\n\n# Analyze bundle size\nvite build --analyze\n```\n\n#### Critical CSS Inlining\n```html\n<!-- Inline critical above-the-fold CSS -->\n<style>\n  /* Mobile-first base styles only */\n  body { font-family: Inter, sans-serif; }\n  header { position: sticky; }\n</style>\n```\n\n#### Asynchronous Stylesheet Loading\n```html\n<!-- Non-critical stylesheets -->\n<link\n  rel=\"preload\"\n  href=\"non-critical.css\"\n  as=\"style\"\n  onload=\"this.onload=null;this.rel='stylesheet'\"\n/>\n```\n\n### 4. Code Splitting\n```jsx\n// Route-based code splitting\nconst Home = lazy(() => import('./pages/Home'));\nconst Booking = lazy(() => import('./pages/Booking'));\n\n// Suspended with loading fallback\n<Suspense fallback={<LoadingSpinner />}>\n  <Home />\n</Suspense>\n```\n\n---\n\n## 🎨 Design System Standards\n\n### Color Palette - WCAG AAA Verified\n\n| Use Case | Color | Code | Contrast Summary\n\n### Do's ✅\n- Use mobile-first CSS approach\n- Test on real devices, not just simulators\n- Provide text alternatives for images (alt text)\n- Ensure keyboard accessibility\n- Implement touch targets ≥44px × 44px\n- Use semantic HTML elements\n- Optimize images with lazy loading\n- Test with screen readers (NVDA, JAWS)\n- Respect `prefers-reduced-motion`\n- Provide focus indicators\n\n### Don'ts ❌\n- Don't rely on color alone to convey meaning\n- Don't disable touch zoom functionality\n- Don't use fixed font sizes\n- Don't forget alt text on images\n- Don't create click-hijacking overlays\n- Don't disable keyboard navigation\n- Don't have text smaller than 12px on mobile\n- Don't use placeholder text as labels\n- Don't auto-play videos with sound\n- Don't create infinite scroll without pagination\n\n---\n\n## 📞 Support & Questions\n\nFor questions about the mobile optimization implementation:\n1.
